# Assessment Design Skill

Florida BEST-aligned assessment design principles for 10th grade Intensive Reading, including STOP protocol, RACE/CER constructed response frameworks, and rubric creation.

## ⚠️ MANDATORY: DISTRICT ALIGNMENT

Before building ANY assessment, you MUST read `mc-question-generation/references/district-planning-guide.md` and follow these rules:

1. **Question Stems**: Use ONLY the district-provided question stem templates. Adapt by filling in blanks with text-specific details. Do NOT invent new stem patterns.
2. **Assessment Limits**: Check what items CAN and CANNOT ask for the target benchmark. Common violations to avoid:
   - R.1.2: Do NOT ask students to define the theme → ask HOW it's developed
   - R.2.2: Do NOT ask to identify the central idea → ask about the SUPPORT for it
   - R.3.1: Do NOT ask to identify type of figurative language → ask how it creates MOOD
   - R.3.4: Do NOT ask to identify type of rhetoric → ask how author USES it
3. **Achievement Levels**: Use Level 3-4 descriptors to calibrate assessment difficulty (not just Level 2 recall)
4. **Part A/B Format**: Include a mix of single-part and two-part questions. Part A = analysis/identification; Part B = text evidence supporting Part A
5. **Analytical Processes**: Ensure questions align with the Planning Card step-by-step processes students are taught

## STOP Protocol (Multiple Choice Design)

STOP is an acronym for analyzing answer choices. Every multiple choice question should have 4 choices that follow this pattern. STOP is taught as Step 3 of the question annotation strategy (CUBES → Find Key Details → STOP) in Days 1-2 and applied independently in Days 5-6 assessments. **For the full STOP teaching guide, anchor charts, practice templates, and ESOL modifications, see `stop-strategy`.**

**Two STOP Modes:**

**Simplified Default (Most Assessments):** Student selects the Proven answer and writes 1-sentence reasoning explaining why it is correct. This is the standard approach for daily work and most assessments.

**Full STOP (Once Per Unit as Teaching Exercise):** Student labels each answer choice with S/T/O/P and writes justification for each label. This deep analysis is used once per unit as a focused teaching and learning exercise.

### S - Silly
- Obviously incorrect
- Easy to eliminate
- May be factually wrong or completely unrelated
- Builds student confidence

**Example:** If asking about theme in a tragedy:
- "The story is funny and lighthearted" (contradicts the passage)

**Teaching Note:** Students learn STOP in Days 1-2 annotation lessons via CUBES question strategy, then apply it independently in Days 5-6 assessments.

### T - Tricky
- Sounds partially correct
- Contains accurate details BUT misses the key concept
- Students who skim might choose this
- Tests deeper understanding

**Example:**
- "The author uses descriptive language" (true, but doesn't answer the theme question)

### O - Opposite
- States the reverse or contrary of the correct answer
- Tests if students truly understand the concept
- May contain similar vocabulary to correct answer

**Example:**
- "Hard work leads to failure" (when the theme is "perseverance leads to success")

### P - Proven
- Correct answer
- Can be proven with text evidence
- **MUST use academic language and vocabulary from the benchmark standard** — the Proven answer should model the kind of precise, benchmark-aligned language students need to use in their own analysis
- Addresses the full scope of the question
- Uses the same terminology found in the benchmark's Common Vocabulary and Achievement Level Descriptors

**Example:**
- "Perseverance in the face of adversity leads to personal growth" (proven in text)

## Constructed Response Strategy (RACE / CER)

**One framework per unit:** Each unit uses either RACE (default) or CER (argument-heavy units). All CR activities, scaffolds, and assessment prompts within that unit use the chosen framework consistently. **For the full RACE teaching guide, tier progression, anchor charts, and ESOL modifications, see `race-strategy`.** For the full CER framework, see `cer-writing-guide`.

Students progress through three tiers of CR complexity across the year:

**Semester 1 - Lite:** Restate + Answer only. This foundation builds clear response structure and the habit of restating the question.

**Semester 2 Start - Standard:** Restate + Answer + Cite. This adds rigor through text evidence with paragraph citations.

**Semester 2 Mid+ - Full:** Complete R-A-C-E or C-E-R response. This represents the complete, sophisticated response expected by year-end.

**CR on assessments:** Every mini-assessment (Days 5-6) includes 1 CR prompt that is always benchmark-aligned. The CR uses the unit's chosen framework at the current tier level.

### RACE Components (Default):

### R - Restate the Question
- Turn the question into a statement
- Use key words from the question
- Creates a clear topic sentence

**Question:** "How does the author develop the theme?"
**Restate:** "The author develops the theme through..."

### A - Answer the Question
- Provide a clear, direct answer
- Use academic vocabulary
- Be specific

**Answer:** "...the character's repeated failures and eventual success."

### C - Cite Text Evidence
- Use quotation marks
- Include paragraph number
- Choose the MOST relevant evidence
- Keep quotes brief (1-2 sentences max)

**Cite:** "In paragraph 12, the text states, 'After three failed attempts, Maria finally succeeded.'"

### E - Explain the Connection
- Connect the evidence back to your answer
- Explain HOW the evidence supports your point
- Use reasoning words: "This shows," "This demonstrates," "This reveals"

**Explain:** "This demonstrates that the author uses the character's journey to show that persistence overcomes obstacles."

### Complete RACE Example:

**Question:** How does the setting contribute to the mood?

**RACE Response:**
The setting contributes to the mood by creating a sense of isolation and unease. In paragraph 5, the author describes "the abandoned cabin sat deep in the forest, miles from any other home." This demonstrates that the author uses the remote location to make readers feel the character's loneliness and vulnerability.

## CER (Claim-Evidence-Reasoning) — For Argument-Heavy Units

CER is used instead of RACE for units focused on argumentation (e.g., R.2.4 Argument, R.2.3 Purpose & Perspective). See `cer-writing-guide` for full details on when to choose CER vs RACE.

**Note:** The word "claim" can be tricky in ELA — explicitly teach that a claim is an interpretation (debatable), not a fact.

### C - Claim
- Your position or interpretation
- Answers the prompt
- Debatable (not just a fact)

### E - Evidence
- Specific facts, quotes, or data from text
- Multiple pieces strengthen argument
- Cited with paragraph numbers

### R - Reasoning
- Explains HOW evidence supports claim
- Makes the thinking visible
- Connects evidence to claim explicitly

## Rubric Design Principles

### For RACE/CER Responses:

**4 Points (Exceeds):**
- Restates question clearly
- Provides thorough answer
- Includes 2+ relevant text quotes with paragraph citations
- Explains connection with sophisticated reasoning

**3 Points (Meets):**
- Restates question
- Answers question directly
- Includes 1 relevant text quote with citation
- Explains connection clearly

**2 Points (Approaching):**
- Partial restatement
- Incomplete answer
- Evidence present but may lack citation or relevance
- Weak or missing explanation

**1 Point (Below):**
- No restatement
- Vague or incorrect answer
- No text evidence or incorrect evidence
- No explanation

**0 Points:**
- No response or completely off-topic

### For Graphic Organizers:

**Completion Points:**
- All rows completed
- Evidence cited with paragraph numbers
- Analysis/explanation in appropriate columns

**Accuracy Points:**
- Evidence supports the claim
- Interpretations align with text
- Benchmark skill correctly applied

**Quality Points:**
- Specific evidence (not generic)
- Thoughtful analysis
- Academic language used

## Question Stems — MANDATORY: Use Planning Card Stems Only

### THE RULE: No Custom-Invented Question Stems

**Every MC question on every assessment MUST use a question stem from the benchmark's Planning Card (found in the "Task Demands and Question Stems" section of each benchmark standard guide in the `benchmarks` skill).** Fill in the blanks with text-specific details (character names, paragraph numbers, specific themes), but the stem structure MUST come from the Planning Card.

**Why this matters:** Planning Card stems mirror how questions are phrased on the FAST PM and state assessments. When students practice with these exact stems, they build familiarity with the language and format they'll encounter on high-stakes tests. Custom-invented stems may test different skills than the benchmark intends.

**How to use:**
1. Load the benchmark standard guide from `benchmarks` skill
2. Go to "Task Demands and Question Stems" section
3. Select stems that match what you want to assess
4. Fill in the blanks with text-specific content
5. Include a mix from Task Demand 1 AND Task Demand 2 (if both exist)
6. Include at least one Part A/Part B question per assessment
7. Tag each question with the task demand it addresses

**DO NOT:** Invent your own question stems. If the Planning Card doesn't have a stem for what you want to assess, the question may not be aligned to the benchmark.

### Generic Fallback Stems (ONLY when no benchmark-specific stems exist)

These are used ONLY for the 20% vocabulary portion or for benchmarks where specific Planning Card stems haven't been loaded:

**Key Ideas & Details:**
- "What is the central idea of paragraph X?"
- "Which detail best supports the theme?"
- "How does the author develop the theme across the text?"

**Craft & Structure:**
- "How does the author's word choice in paragraph X contribute to...?"
- "What is the purpose of [literary device] in this passage?"
- "How does the structure of the text contribute to its meaning?"

**Integration of Knowledge:**
- "How does the author use [evidence/reasoning] to support their claim?"
- "Which statement from the text best supports the author's argument?"

### 20% Vocabulary MC Stems

These questions test bellringer/unit vocabulary in context. They count toward the vocabulary portion of the 80/20 MC split.

- "What does [word] most likely mean as used in paragraph X?"
- "Which word could replace [word] without changing the meaning?"
- "How does the context help determine the meaning of [word]?"

## Assessment Formatting Guidelines

### Multiple Choice:
- Question stem in **bold**
- 4 answer choices (A-D)
- Use ○ (not filled circles) for answer bubbles
- Include paragraph reference: "(paragraph X)"
- Leave space for STOP analysis

**Format:**
```
**Question stem here?** (paragraph 5)
○ A. Silly choice
○ B. Tricky choice
○ C. Opposite choice
○ D. Proven choice

STOP Analysis:
A is: _____ B is: _____ C is: _____ D is: _____
```

### RACE/CER Response:
- Question in bold
- Lines provided for each component (RACE: R-A-C-E; CER: C-E-R)
- Paragraph citation space
- Scoring rubric on same page

**RACE Format:**
```
**Question here?**

R (Restate): _________________________________

A (Answer): __________________________________

C (Cite Evidence): ____________________________
   "_______________________" (paragraph ____)

E (Explain): _________________________________
____________________________________________

Score: _____ / 4 points
```

## Differentiation for Assessments

### ESOL Modifications:
- Word bank for vocabulary
- Sentence frames for RACE or CER
- Translated key terms (if appropriate)
- Extended time
- Read-aloud option

### Scaffolds:
- Paragraph numbers highlighted in text
- STOP labels provided (student just matches)
- Sentence starters for explanation
- Graphic organizer to plan before writing

### Extensions:
- Multi-paragraph response
- Compare two texts
- Evaluate author's effectiveness
- Create original example

## Best Practices

1. **Align to benchmark**: Use exact language from Florida BEST standards
2. **Text-dependent**: Questions must require reading the text
3. **Grade-appropriate**: Complexity matches 10th grade expectations
4. **Fair and unbiased**: Avoid cultural assumptions or trick questions
5. **Clear and concise**: Questions should be understandable
6. **Adequate space**: Provide enough room for student responses
7. **Rubric transparency**: Students should know expectations before starting
8. **Page budget**: Assessment materials must fit within the daily page budget (1 front/back page per day)
9. **AI independence**: AI is not used during assessments. All assessment work is done independently without AI assistance.

## Common Pitfalls to Avoid

❌ Questions answerable without reading text
❌ Multiple correct answers possible
❌ Vocabulary in question harder than in text
❌ All STOP choices too obvious or too tricky
❌ CR questions asking for opinion instead of text analysis
❌ No paragraph citations required
❌ Rubrics unclear or subjective

## Assessment Types for IR

### Daily Formative:
- Exit tickets with 1 CR question (tier-appropriate, using ACE shortened form)
- Quick STOP analysis using Simplified mode (2-3 questions)
- Organizer spot-checks

### Mini-Assessments (Days 5-6):
- **6-8 multiple choice** (STOP-aligned, Simplified mode):
  - **80% benchmark-aligned** (5-6 questions testing the unit benchmark skill)
  - **20% vocabulary** (1-2 questions testing bellringer/unit vocabulary in context)
- **1 constructed response** (tier-appropriate, always benchmark-aligned):
  - Uses unit's chosen framework (RACE or CER) at current tier level
  - Prompt directly assesses the unit benchmark — never a generic or vocabulary-only prompt
- 20-minute completion time
- Must fit within daily page budget (1 front/back page)

### Unit Assessments:
- **10-12 multiple choice** (STOP-aligned, Simplified mode):
  - **80% benchmark-aligned** (8-10 questions)
  - **20% vocabulary** (2 questions)
- **1-2 constructed responses** (tier-appropriate, always benchmark-aligned)
- Mirrors FSA format
- Must fit within daily page budget (1 front/back page)

### Full STOP Teaching Exercise (Once Per Unit):
- Deep analysis activity using Full STOP protocol
- 2-4 multiple choice questions
- Students label each choice S/T/O/P and justify each
- Conducted in class as focused learning exercise

## Output Specification

Assessment design produces assessments in the following formats and structures:

### Assessment Document Format
- **File type**: .docx (Word document) for print/digital delivery
- **Components**:
  1. **Header** (standard/benchmark code, unit name, date, class period)
  2. **Instructions** (purpose, time limit, format overview)
  3. **Multiple choice section** (6-8 questions: 80% benchmark / 20% vocabulary, STOP-aligned distractors)
  4. **Constructed response section** (1 CR prompt using unit's RACE or CER framework, always benchmark-aligned)
  5. **Scoring rubric** (aligned to 4-point or completion-based scale)
  6. **Answer key** (for teacher records)

### Question Bank Organization
- **Format**: CSV or Excel workbook
- **Columns**: Benchmark code, question type (MC/CR), question stem, correct answer, distractor options, difficulty level, date created
- **Purpose**: Reusable question bank for assessment assembly

### Rubric Documentation
- **Format**: Detailed rubric with descriptors for each performance level
- **Rows**: Performance levels (Exceeds/Meets/Approaching/Below/Incomplete)
- **Columns**: Point value, indicator criteria, exemplars/non-examples
- **File type**: Shareable via Google Docs or .docx

## Integration with Data Pipeline

After assessments are administered, student response data flows through this pipeline:

1. **Assessment administered** → Raw response data collected
2. **student-data-processor** → Cleans and standardizes raw assessment data
3. **data-quality-checker** → Validates completeness, flags missing/anomalous scores
4. **growth-analyzer** → Calculates growth metrics and risk scores per student
5. **report-builder** → Assembles growth reports, intervention recommendations, and stakeholder updates

Assessment design ensures alignment with all downstream tools by including:
- Benchmark codes on every question (for data labeling)
- RACE/CER structure (for automated scoring and reporting)
- Consistent rubric scales (for comparison across units)
