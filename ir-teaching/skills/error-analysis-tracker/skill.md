# Error Analysis Tracker — 10th Grade Intensive Reading

## Purpose

Error analysis turns wrong answers into instructional data. Instead of just marking answers right or wrong, students (and teachers) identify WHY an answer was wrong — and that diagnosis drives what happens next. A student who consistently falls for Tricky answers needs different instruction than one who Misreads questions or Rushes through.

**The core principle:** Wrong answers aren't random. They fall into patterns. Once you see the pattern, you can fix it.

**Where this fits:** The error codes are defined in `attack-the-passage`. This skill is about the SYSTEM — how to collect, aggregate, analyze, and act on error data across texts and across the class.

---

## Error Type Codes

| Code | Meaning | What Happened | Looks Like |
|------|---------|---------------|-----------|
| **S** | Fell for Silly | Selected an answer with no text support at all | Chose something that "sounds right" but isn't in the passage |
| **T** | Fell for Tricky | Selected an answer that's partially true or sounds good but doesn't answer THIS question | Chose something related to the text but not the best answer |
| **O** | Chose Opposite | Selected an answer that says the reverse of what the text says | Confused "supports" with "undermines," misread tone direction |
| **MR** | Misread Question | Answered a different question than what was asked | Missed a key word in the stem ("NOT," "EXCEPT," specific paragraph) |
| **R** | Rushed | Didn't read carefully, guessed, or ran out of time | Pattern: wrong answers cluster at end of test, or justifications are blank/vague |
| **V** | Vocabulary | Unfamiliar word in stem or answer choices caused confusion | Wrong because of a word they didn't know, not a reading skill gap |
| **NE** | No Evidence | Didn't use text evidence to select answer | Justification is opinion-based or says "I just thought" instead of citing text |

## Teacher-Level Aggregation

### Class Error Audit (After Each Text)

After scoring a text's cold read, tally error types across the class. This takes 10-15 minutes and drives the next day's instruction.

```
CLASS ERROR AUDIT — [Text Title] — [Period]

Error Type | Tally | Count | % of All Errors | Action
-----------|-------|-------|-----------------|-------
S (Silly)  |       |       |                 |
T (Tricky) |       |       |                 |
O (Opposite)|      |       |                 |
MR (Misread)|      |       |                 |
R (Rushed) |       |       |                 |
V (Vocab)  |       |       |                 |
NE (No Ev) |       |       |                 |

Total Errors: ___
Class Avg Score: ___/10
```

### How to Use the Audit

| If Top Error Is... | What It Means | Instructional Response |
|--------------------|--------------|----------------------|
| **S (Silly)** > 25% | Students aren't reading answer choices carefully | Reteach: "Read ALL four choices before selecting." Practice full STOP labeling. |
| **T (Tricky)** > 30% | Students can't distinguish "partially true" from "best answer" | This is the #1 error. Reteach: "Does it answer THIS question?" Model Tricky vs. Proven comparison. Pull small group for guided elimination. |
| **O (Opposite)** > 15% | Students are confusing direction/tone of the text | Reteach annotation: "Is the author FOR or AGAINST? Does the character GROW or DECLINE?" Focus on margin notes for tone/direction. |
| **MR (Misread)** > 20% | Students aren't reading question stems carefully | Reteach Phase 1 C-U-B on questions. Slow down. Have students restate the question in their own words before answering. |
| **R (Rushed)** > 20% | Pacing problem — students run out of time or don't try | Adjust time limits. Teach time management: "Spend 2-3 min per question, not 30 sec." Check if text is too complex. |
| **V (Vocab)** > 15% | Vocabulary gaps are blocking comprehension | Add vocabulary pre-teach to Day 1 bellringer. Provide glossary during cold read. Flag specific words for reteach. |
| **NE (No Evidence)** > 20% | Students aren't connecting answers to text | Reteach Phase 2 margin notes. Require Q# in margins. Model: "Where in the text does it say that?" |

### Per-Question Error Analysis

For deeper diagnosis, track which questions had the highest error rates:

```
PER-QUESTION ERROR BREAKDOWN — [Text Title]

Q# | Benchmark | % Correct | Most Common Wrong Answer | Error Type | Notes
---|-----------|-----------|------------------------|------------|------
1  |           |           |                        |            |
2  |           |           |                        |            |
...
10 |           |           |                        |            |

Questions with < 50% correct: _______________

```

**Action protocol:** Any question with < 50% correct gets reviewed whole-class on Day 3 or Day 4. The teacher models the full STOP elimination for that question.

## Tracking Error Patterns Across Texts (Unit-Level)

### Unit Error Trend

After all 4 texts, compare error patterns to see if instruction is working:

```
UNIT ERROR TREND — [Period]

Error Type | Text 1 | Text 2 | Text 3 | Text 4 | Trend
-----------|--------|--------|--------|--------|------
S          |   %    |   %    |   %    |   %    | ↑↓→
T          |   %    |   %    |   %    |   %    | ↑↓→
O          |   %    |   %    |   %    |   %    | ↑↓→
MR         |   %    |   %    |   %    |   %    | ↑↓→
R          |   %    |   %    |   %    |   %    | ↑↓→
V          |   %    |   %    |   %    |   %    | ↑↓→
NE         |   %    |   %    |   %    |   %    | ↑↓→
```

**What to look for:**
- **↓ Decreasing S/T/O:** STOP instruction is working. Students are getting better at elimination.
- **→ Flat T:** Tricky remains stubborn. Need more guided practice distinguishing Tricky from Proven.
- **↑ Increasing MR:** Students may be getting overconfident and skipping stem analysis. Reinforce Phase 1.
- **↑ Increasing R:** Texts may be getting harder, or stamina is declining. Check text complexity progression.

### Student-Level Chronic Error Identification

Flag students who show the same error type across 3+ texts:

```
CHRONIC ERROR REPORT

Student | Error Type | Text 1 | Text 2 | Text 3 | Text 4 | Intervention
--------|-----------|--------|--------|--------|--------|-------------
        | T         | 4/10   | 3/10   | 5/10   | 4/10   | Pull for STOP reteach
        | MR        | 3/10   | 4/10   | 3/10   | 2/10   | C-U-B reteach, slow stem reading
        | V         | 2/10   | 3/10   | 4/10   | 3/10   | Vocab pre-teach, glossary access
```

**Threshold for "chronic":** Same error type is top error for 3+ texts, OR same error type accounts for > 40% of a student's total errors across the unit.

## Connecting Error Data to Instruction

### Error → Skill → Reteach Map

| Error Code | Underlying Skill Gap | Reteach Skill | Reteach Activity |
|-----------|---------------------|--------------|-----------------|
| S | Not reading answer choices | `stop-strategy` | Full STOP labeling practice (label ALL 4 choices) |
| T | Can't distinguish partial truth from best answer | `stop-strategy` + `question-stem-analysis` | Side-by-side comparison: "Both B and C mention the character. Which answers THIS question?" |
| O | Misreading tone, direction, or meaning | `cubes-annotation` | Re-annotate passage focusing on tone markers, direction words |
| MR | Not reading question carefully | `attack-the-passage` Phase 1 | C-U-B reteach; restate question in own words before answering |
| R | Time management or engagement | `test-prep-unit` | Adjust cold read time; chunk into 5+5 questions; engagement check |
| V | Vocabulary gaps | `vocabulary-instruction` | Pre-teach Tier 2/3 words; provide glossary; context clue bellringers |
| NE | Not connecting to text evidence | `attack-the-passage` Phase 2 | Margin note reteach; require Q# in margins; model evidence selection |

### Small Group Pull Schedule

Based on error audit data, pull groups during independent center time:

| Day | Who to Pull | What to Reteach | Time |
|-----|------------|-----------------|------|
| Day 2 | Students with MR errors | C-U-B on questions, restate stems | 10 min |
| Day 3 | Students with T errors | STOP elimination: Tricky vs. Proven | 15 min |
| Day 3 | Students with NE errors | Phase 2 margin notes, evidence selection | 10 min |
| Day 4 | Students with V errors | Vocabulary from this text + context clues | 10 min |

## ESOL Connections

Error analysis is especially useful for ESOL students because it distinguishes language barriers (V errors) from reading skill gaps (T, O, NE errors).

**Key diagnostic question:** "Is this student getting questions wrong because of vocabulary/language access, or because of a reading comprehension strategy gap?"

- **High V errors** → Language support: glossaries, vocabulary pre-teach, bilingual resources
- **High T/O/NE with low V** → Strategy support: STOP practice, annotation, evidence connection
- **High MR** → Stem language support: pre-teach question vocabulary ("infer," "primarily," "contribute")

See `esol-core` for full modification strategies.
