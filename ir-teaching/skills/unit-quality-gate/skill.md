# Unit Quality Gate

## Purpose

Verify a unit is actually complete before it goes to students. Evidence before claims — never
say a unit is done without running this checklist first.

## When to Use

- Automatically: at end of every unit-builder-protocol build
- Manually: after any revision, before printing, before uploading to OneDrive

## The Checklist

Run each item in order. Report PASS or FAIL with specifics for each.

### 1. Benchmark Alignment (HIGHEST PRIORITY — check this first)

- [ ] Benchmark standard guide was loaded before building (BENCHMARK-FIRST mandate)
- [ ] **Day 1 TL explicitly teaches the benchmark:** standard stated on board, benchmark vocabulary defined, analytical process previewed
- [ ] **Day 2 TL teaches the analytical process:** step-by-step from benchmark guide, Planning Card question stems introduced
- [ ] **All MC questions use Planning Card question stems** — verify each stem matches a stem from the benchmark's "Task Demands and Question Stems" section (no custom-invented stems)
- [ ] **MC answer choices use benchmark vocabulary** — Proven answers use academic language from the standard
- [ ] **Bellringers include benchmark vocabulary:** at least 4-6 of 18 words come from the benchmark's Common Vocabulary list
- [ ] **Organizer column headers map to benchmark analytical process steps** — not generic columns
- [ ] **CR prompt directly assesses the benchmark** using benchmark language, not a generic writing prompt
- [ ] **Student packet directions use benchmark vocabulary** — students encounter benchmark terms daily in their task directions

**FAIL condition:** Any MC question uses a custom-invented stem instead of a Planning Card stem. Any unit where Day 1-2 does not explicitly teach the benchmark. Any bellringer set with zero benchmark vocabulary words.

### 2. Deliverables Exist (all 4 required)

- [ ] `[Text]_TeacherPlan_[MM-DD].docx` exists
- [ ] `[Text]_StudentPacket_[MM-DD].docx` exists
- [ ] `[Text]_AnswerKey_[MM-DD].docx` exists
- [ ] `[Text]_FeedbackForm_[MM-DD].docx` exists
- [ ] `_ChangeLog.txt` exists

**FAIL condition:** Any of the 4 core .docx files is missing.

### 3. File Synchronization

- [ ] All 4 files use the same date suffix
- [ ] Vocabulary is identical across TeacherPlan, StudentPacket, and AnswerKey (same 18 words, same days)
- [ ] Day structure matches (6 days in all files, same day labels)
- [ ] Organizer structure matches (same rows and column headers in all files)
- [ ] CR framework is consistent (RACE throughout OR CER throughout — never mixed)

**FAIL condition:** Any mismatch between files.

### 4. Bellringers (Days 1-6)

- [ ] StudentPacket has a bellringer section for every day (Days 1-6)
- [ ] Each bellringer has exactly 3 vocabulary words from the unit text
- [ ] Each bellringer has 2 MC items + 1 written response
- [ ] TeacherPlan has complete bellringer answer keys for all 6 days
- [ ] AnswerKey has bellringer answers with definition, context clues, and acceptable variations

**FAIL condition:** Missing bellringer on any day, or missing answer key.

### 5. Organizer (Days 3-4)

- [ ] Organizer has I Do row (pre-filled as exemplar with text citations)
- [ ] Organizer has We Do row (partially complete or guided)
- [ ] Organizer has You Do w/ Partner row (sentence starters present)
- [ ] Organizer has You Do row (minimal or no scaffold)
- [ ] Organizer columns align to benchmark card steps
- [ ] StudentPacket organizer matches TeacherPlan organizer structure

**FAIL condition:** Missing scaffold rows or organizer structure doesn't match benchmark.

### 6. Assessment (Days 5-6)

- [ ] StudentPacket has Day 5-6 assessment section
- [ ] Assessment includes MC questions with STOP format
- [ ] Assessment includes CR writing task (RACE or CER)
- [ ] AnswerKey has complete answers for all MC items with STOP justifications
- [ ] AnswerKey has exemplar RACE/CER response with text citations

**FAIL condition:** Missing assessment or missing complete answer key.

### 7. Feedback Form

- [ ] FeedbackForm includes benchmark-specific checkboxes (not generic)
- [ ] Day 4 organizer checkpoint is embedded in StudentPacket
- [ ] Day 6 CR checkpoint is embedded in StudentPacket
- [ ] Student Action Plan section is present in FeedbackForm

**FAIL condition:** Generic (non-benchmark) feedback form, or missing embedded checkpoints.

### 8. ESOL (if applicable)

- [ ] If ESOL students are in the class: sentence frames are present in StudentPacket
- [ ] If ESOL students are in the class: word banks are present where appropriate
- [ ] TeacherPlan has 2+ labeled ESOL strategies per day

**FAIL condition:** ESOL modifications absent when ESOL students are in class roster.

### 9. Brand Compliance (spot check)

- [ ] No emojis in section headers
- [ ] Day headers use format "DAY [#]: [Title]"
- [ ] Section headers: BELLRINGER, TEACHER-LED, INDEPENDENT (exact caps, no variations)
- [ ] Font is Arial throughout (no mixed fonts)

**FAIL condition:** Brand violations that would confuse students or look unprofessional.

## Reporting

**If ALL items pass:**
```
QUALITY GATE: PASSED ✓

All 9 checks passed. Unit is ready.
→ Next: run unit-distribution to ship it, or unit-reviewer for a fresh-eyes check.
```

**If any item fails:**
```
QUALITY GATE: FAILED ✗

Failed checks:
- [3. Bellringers] Day 4 bellringer answer key is missing from TeacherPlan
- [6. Feedback Form] Day 4 organizer checkpoint not embedded in StudentPacket

Do NOT call this unit done. Fix the above before distributing.
```

**Never express satisfaction or say the unit is complete if any check fails.**
